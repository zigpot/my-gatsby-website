Making That Weird and Funny Floating Face Effect in Python

Fariz A.
Thu Sep 23 2021

At first I thought this was one of Tiktok's newest effect. However it wasn't. This effect was achieved in a rather manual way by it's creators, usually using Adobe Premiere and/or other dedicated video/image editing softwares. I'm not used with those kinds of software. Plus, since it's seems pretty easy, why not making it ourselves? After all, this seems like a simple challenge. We can fire up our Python editor. Also we are going to use OpenCV.

For our example we are going to use this video from TikTok:

We gonna have to detect a face. So, in this case we'll be using dlib, a C++ library notorious for it's machine learning components. We are also going to use imutils, for basic image manipulation. Also for matrix array manipulation, we'll ask our old friend NumPy, a helping hand.

Now that we have the frame of choice, we need to find a face, and extract it. For that, we'll be using an 81 points of facial landmarks detection model, as this one also includes the forehead (clue: most don't).

#face extraction

gray = cv2.cvtColor(frame, cv2.COLOR_BGR2GRAY)
out_face = np.zeros_like(frame)

detector = dlib.get_frontal_face_detector()

   feature_mask = feature_mask.astype(np.bool)
   feature_mask = feature_mask * np.uint8(255)

   new_img = cv2.cvtColor(image, cv2.COLOR_BGR2BGRA)
   new_img[:, :, 3] = feature_mask
   contours = cv2.findContours(feature_mask, cv2.RETR_EXTERNAL, cv2.CHAIN_APPROX_SIMPLE)
   contours = contours[0] if len(contours) == 2 else contours[1]
   cntr = contours[0]
   xpos,ypos,w,h = cv2.boundingRect(cntr)

   face = new_img[ypos:ypos+h, xpos:xpos+w]

   cv2.waitKey(0)
   cv2.destroyAllWindows()

face_height, face_width, _ = face.shape

The program above does the following:

  1. Detect face (called Region of Interest or ROI), this creates a feature mask.
  2. Blacken the area outside the ROI using the feature mask.
  3. Crop to the ROI.

Face cropped to it's surrounding.

This crop region is square and doesn't look nice with the black areas around. We want a nice facial shape. To achieve this, we convert the face to a four channel data type (RGB + Alpha also known as the transparency). The feature mask is turned into a boolean matrix, it's black area gets zero-ed using a straightforward logic operation, and finally appended to the image as the alpha channel. In other words, the black area becomes transparent, while the white area is the only opaque area.

Still square, but now it's transparent outside the ROI

Now that we have the face of interest, we want to put it somewhere on the video and move it until it reaches it's origin, where it was extracted from. With origin I mean both the spatial (location) and temporal (frame) origin.

We want Flo's face to reach where it came from, on the exact source frame.
